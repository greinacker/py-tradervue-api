Python TraderVue API¶

This is a Python implementation of the TraderVue API.

Below is a small example usage:

import datetime
import tradervue
tv = tradervue.TraderVue(username, password, user_agent)
trades = tv.get_trades(symbol = 'OEX', startdate = datetime.date(2015, 9, 1))
for t in trades:
  print "Trade %s Profit/Loss: $%s" % (t['id'], t['gross_pl'])

Construct a TraderVue instance.

Parameters:
  • username (str) – the TraderVue username
  • password (str) – the TraderVue password
  • user_agent (str) – the user agent to use in requests. Should be something like: MyApp (your@email.com)

create_trade(symbol, notes=None, initial_risk=None, shared=False, tags=[], return_url=False)¶

Create a new trade. This is the equivalent of the ‘New Trade’ feature on the website.

Parameters:
  • symbol (str) – The symbol for the trade
  • notes (str or None) – Any notes for the trade. Can include Markdown syntax.
  • initial_risk (float or None) – The initial risk for the trade
  • shared (bool) – True if this trade should be shared with other TraderVue users
  • tags (list) – A list of tags to be applied to this trade. Each tag should be a string.
  • return_url (bool) – If set to True, the return value will be the value of the Location header. If False the trade ID is returned.
Returns:

The new trade ID if return_url is False or the Location URL if it is True. None is returned if an error occurs.
Return type:

str or None

get_trades(symbol=None, tag_expr=None, side=None, duration=None, startdate=None, enddate=None, winners=None, max_trades=25)¶

Query for trades matching the specified criteria.

All arguments to this method are optional. If not specified, they are not part of the query.

The list returned from this method contains dict objects which have fields as defined in the TraderVue Trade Documentation.

Parameters:
  • symbol (str or None) – Find trades on this symbol
  • tag_expr (str or None) – Find trades matching this tag expression. Read more about tag expressions in this blog entry.
  • side (str or None) – Find trades matching the specified side. Must be one of the following values: 'Long' or 'Short'.
  • duration (str or None) – Find trades matching the specified duration. Must be one of the following values: 'Intraday' or 'Multiday'.
  • startdate (datetime or None) – Find trades occuring on or after the specified time
  • enddate (datetime or None) – Find trades occuring on or before the specified time
  • winners (bool or None) – Find trades where the P&L is positive (or negative for a False value).
  • max_trades (int or None) – Return at most the specified number of trades
Returns:

a list of trades matching the specified critiera or None if an error is encountered
Return type:

list or none

import_executions(executions, account_tag=None, tags=None, allow_duplicates=False, overlay_commissions=False, import_retries=3, wait_for_completion=False, wait_retries=3, secs_per_wait_retry=15)¶

Import the specified trade executions.

Parameters:
  • executions (list) – The executions to import. This should be a list of dicts. Each dict should have keys as specified in the TraderVue Import Documentation.
  • account_tag (str or None) – An account tag to use when importing. If None, no account tag is used.
  • tags (list or None) – A list of tags to be applied to this trade. The list values should be strings. IF None, no tags are applied to the trade.
  • allow_duplicates (bool) – set this to True if you wish to disable Tradervue’s automatic duplicate-detection when importing this data.
  • overlay_commissions (bool) – set this to True to run this import in commission-overlay mode; no new trades will be created, and existing trades will be updated with commission and fee data. See the Tradervue help article for more details.
  • import_retries (int) – TraderVue allows only one import at a time. If this method is invoked while TraderVue is busy, the import will be retried up to this many times before returning None.
  • wait_for_completion (bool) –

    If True, this method will block until the import has been processed by TraderVue. In this case, the import success/failure information will be the return value from this method. Details on that data structure are available in the TraderVue Import Documentation.

  • wait_retries (int) – The number of times to poll the import status before giving up and returning None.
  • secs_per_wait_retry (int) – The poll interval in seconds to query import status.
Returns:

If wait_for_completion is True returns the import status dict or None on error. Otherwise returns True on success or False if an error occurs.
Return type:

dict or None
Raises:
  • ValueError – if executions is empty
  • TypeError – if executions or tags are not list objects
